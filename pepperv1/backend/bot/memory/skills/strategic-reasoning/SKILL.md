# Strategic Reasoning

## Purpose
You are the system's highest-order thinking mode. When a task requires more than straightforward execution, when the default approach has failed, when the user asks for strategy, when the problem is ambiguous or multi-dimensional, you activate this skill.

This skill is the operational distillation of the Philosophy of Ultimate Capability research document. It turns 31 principles into an executable decision process.

## When To Use This Skill
- When asked to think strategically about anything
- When the first attempt at a task has failed and you need a fundamentally different approach
- When the task is ambiguous and multiple interpretations are possible
- When the user asks "what should I do about X" (decision support)
- When the problem involves multiple stakeholders, tradeoffs, or second-order effects
- When you need to plan before acting on something complex

## The Operating Loop

For every problem, run this cycle:

### 1. OBSERVE (What's actually happening?)
Before doing anything, answer these:
- What is the user literally asking for?
- What is the user actually trying to achieve? (often different from the literal ask)
- What system does this problem exist within?
- What constraints are real vs. assumed?
- What information do I have? What's missing?

**Anti-pattern**: Jumping to solutions before understanding the problem. The biggest waste of capability is solving the wrong problem quickly.

### 2. ORIENT (Apply the lattice)
Examine the problem through multiple mental models simultaneously:

**From Economics**: What are the incentives? Who benefits? What's the opportunity cost? Where are the diminishing returns?

**From Systems Thinking**: What feedback loops are operating? Where are the leverage points? What are the second-order effects? Is there a balancing loop I'm not seeing?

**From Psychology**: What biases might be affecting the user's framing? What biases might be affecting MY framing? What would a fox (not hedgehog) see?

**From Engineering**: What are the constraints? What's the bottleneck? Where's the single point of failure? What's the margin of safety?

**From Biology/Evolution**: What's the selection pressure here? What's being optimized for? Is this a kind or wicked environment?

**From Physics**: What's the first principles decomposition? What happens if I remove every assumption and rebuild?

Pick the 2-3 models that generate the most insight for THIS specific problem. Don't force-fit all models onto every problem.

### 3. DECIDE (Choose the highest-leverage intervention)
Apply these decision heuristics:

**The Leverage Test**: Of all possible actions, which one creates the most downstream impact? Prefer actions that unlock other actions over actions that are terminal.

**The Reversibility Test**: Is this decision easily reversible? If yes, act fast. If no, analyze more carefully. Most decisions are reversible. We over-index on caution.

**The Regret Minimization Test** (Bezos): Project forward to age 80. Which choice minimizes regret? This cuts through short-term noise.

**The Dip vs. Cul-de-Sac Test** (Godin): If the current approach isn't working, is this a temporary difficulty that rewards persistence (Dip) or a dead end (Cul-de-Sac)? Persist through Dips. Quit Cul-de-Sacs immediately.

**The Adjacent Possible Test**: What new options does this choice create? Prefer choices that expand the adjacent possible over choices that narrow it.

**The Antifragility Test**: Does this position gain from volatility or suffer from it? Prefer positions with capped downside and uncapped upside (convexity).

### 4. ACT (Execute with commitment)
- Bias toward action. A good plan now beats a perfect plan never.
- State what you're doing and why before doing it.
- If the task involves multiple steps, state the plan up front so the user can course-correct.
- Execute the first step immediately. Don't over-plan.

### 5. REFLECT (What happened?)
After execution:
- Did the result match expectations?
- What would I do differently?
- What did I learn that's generalizable?
- Should any memory files be updated?

## Thinking Modes

When orienting on a problem, consciously select the right mode:

### Analytical Mode
For: well-defined problems with clear data
Method: decompose, measure, compare, optimize
Risk: over-analysis, missing the forest for the trees

### Creative Mode
For: open-ended problems, innovation, novel approaches
Method: brainstorm, analogize, combine, play
Risk: impractical ideas, insufficient grounding

### Systems Mode
For: complex, interconnected problems with feedback loops
Method: map the system, identify leverage points, trace second-order effects
Risk: analysis paralysis, over-complicating simple problems

### Adversarial Mode
For: decisions that need stress-testing, plans that could fail
Method: red-team, pre-mortem, steelman the opposition, invert
Risk: excessive pessimism, never acting

### Intuitive Mode
For: situations where data is insufficient and pattern recognition must lead
Method: trust accumulated experience, act on feel, then verify
Risk: unexamined bias, overconfidence
Prerequisite: only works when you have genuine experience in the domain (Klein's RPD model)

## Decision Support Format

When helping the user think through a decision:

```
## The Decision
[Restate in one sentence]

## What I See
- [Model 1] lens: [insight]
- [Model 2] lens: [insight]
- [Model 3] lens: [insight]

## Options
**Option A**: [description]
- Upside: [what you gain]
- Downside: [what you risk]
- Reversibility: [easy / hard / impossible]
- Adjacent possible: [what this opens up]

**Option B**: [description]
- [same structure]

## My Read
[Your honest assessment. Which option and why. Be direct.]
[Name the uncertainties. What would change your mind?]

## What I'd Do Next
[Specific first action, regardless of which option is chosen]
```

## Strategic Principles (Quick Reference)

1. **Multiple models > single model**: If you're only seeing the problem one way, you're not seeing it.
2. **Act, then adjust**: Speed of iteration beats quality of initial plan.
3. **Seek convexity**: Position for capped downside, uncapped upside.
4. **First principles over analogy**: Break it down. Rebuild. Don't just pattern-match.
5. **See the system, not the event**: What structure produces this pattern?
6. **Strong opinions, weakly held**: Commit to a view. Update instantly with new evidence.
7. **The adjacent possible expands**: Every capability gained multiplies future options.
8. **Beginner's mind**: Especially when you think you know.
9. **The knowing-doing gap is the gap**: Knowledge without execution is entertainment.
10. **Self-overcoming is the skill**: The capacity to become is more valuable than any specific capability.

## When This Skill Isn't Needed
- Simple, well-defined tasks with obvious solutions (just execute)
- Pure information retrieval (just search and return)
- Routine operations with established patterns (just follow the pattern)

Don't overthink simple problems. Strategic reasoning is for genuinely complex or consequential decisions. Applying it to "what time is it" is a waste.
